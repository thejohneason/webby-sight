Lesson 1

What is HTML?

HTML stands for HyperText Markup Language. It's not a programming language — it's a markup language that describes the structure of a web page using elements.

An element usually has an opening tag, some content, and a closing tag: 

<p>This is a paragraph.</p>

The browser reads the tags and decides how to display the content. Tags describe meaning (this is a paragraph, this is a heading, this is a link) and the browser handles the visual result.

Lesson 2

The basic page structure

Every HTML page follows the same skeleton:

<!DOCTYPE html>
<html>
  <head>
    <title>My Page Title</title>
  </head>
  <body>
    <h1>Welcome!</h1>
    <p>This is the content people see.</p>
  </body>
</html>
  • <!DOCTYPE html> tells the browser this is a modern HTML document.
  • <html> wraps the entire page.
  • <head> holds information about the page (title, styles, metadata) — nothing in here is visible on the page itself.
  • <body> holds everything the visitor actually sees.

Lesson 3

Text elements

Headings come in six sizes, <h1> (biggest/most important) through <h6> (smallest).

<h1>Main title</h1>
<h2>Section heading</h2>
<p>A normal paragraph of text. You can make words <strong>bold</strong> or <em>italic</em>.</p>
<br> <!-- forces a line break -->
<hr> <!-- draws a horizontal line -->

Use headings to reflect real document structure (one <h1> per page, then <h2> for major sections, <h3> for subsections) rather than just picking whichever looks biggest.

Lesson 4

Links and images

Links use <a> (anchor) with an href attribute pointing to a destination:

<a href="https://example.com">Visit Example</a>

Images use <img>, which is self-closing (no separate closing tag) and requires src (the image location) and alt (a text description for accessibility and for when the image fails to load):

<img src="cat.jpg" alt="A sleeping orange cat">
Since you're deploying to GitHub Pages, keep image paths relative (e.g. images/cat.jpg) and upload the images alongside your HTML file in the same repository.

Lesson 5

Lists

Unordered (bulleted) lists use <ul>, ordered (numbered) lists use <ol>. Each item is an <li>:

<ul>
  <li>Milk</li>
  <li>Eggs</li>
</ul>

<ol>
  <li>Preheat the oven</li>
  <li>Mix ingredients</li>
</ol>

Lesson 6

Attributes, class, and id

Attributes live inside the opening tag and provide extra information, like href on links or src/alt on images. Two attributes you'll use constantly are class and id: 

<p id="intro">Only one element on the page should have this exact id.</p>
<p class="highlight">Many elements can share the same class.</p>

Neither does anything visual by itself — they're hooks that CSS (for styling) and JavaScript (for behavior) use to target specific elements. Think of class as a label you can reuse on many elements, and id as a unique name for exactly one element.

Lesson 7

div and span: generic containers

<div> is a generic block-level container (it takes up its own line) used to group chunks of content for styling or layout. <span> is the inline version, used to wrap a small piece of text inside a sentence. 

<div class="card">
  <h3>Card title</h3>
  <p>Some <span class="price">$9.99</span> item.</p>
</div>

They carry no meaning on their own — they exist purely to group things, which is exactly why the next lesson matters.

Lesson 8

Tables

Tables are built from <table>, rows (<tr>), header cells (<th>), and data cells (<td>):

<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>
  <tr>
    <td>Ada</td>
    <td>28</td>
  </tr>
</table>
Try it
Lesson 9

Forms

Forms collect input from visitors. A basic text form:

<form>
  <label for="name">Your name:</label>
  <input type="text" id="name" name="name">
  <button type="submit">Send</button>
</form>
On a static GitHub Pages site, a plain <form> has nowhere to submit to (there's no server behind the page). To actually collect submissions you'd need a third-party form service — that's a topic for later, but the HTML itself works the same way either way.

Lesson 10

Semantic HTML

Modern HTML includes tags that describe the role of a section, not just its box shape. Using them instead of endless <div>s helps browsers, screen readers, and search engines understand your page: 

<header>Site title / logo</header>
<nav>Navigation links</nav>
<main>
  <article>A self-contained piece of content, like a blog post</article>
  <section>A thematic grouping within the page</section>
</main>
<footer>Copyright, links, etc.</footer>

Lesson 11

Next steps: deploying to GitHub Pages

Once you're happy with a page, GitHub Pages will host it for free straight from a repository:

  1. Create a GitHub repository (public).
  2. Add your HTML file to it and name your homepage index.html.
  3. Go to the repository's Settings → Pages and set the source to your main branch.
  4. GitHub gives you a URL like https://yourusername.github.io/your-repo/.
Since the page is public, avoid putting real names, emails, addresses, or anything private directly in the HTML or in comments — anyone can view your source.
